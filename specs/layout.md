# Layout

## Project Root Detection

The tool finds project root by searching upward for:

1. `.fa/` directory
2. `.git/` directory

## .fa/ Directory Layout

```
.fa/
├── tasks/                          # Task storage
│   ├── 1-03-25-first-task/
│   │   ├── task.md                 # Task description
│   │   ├── task.json               # Task metadata
│   │   └── memory-1.md             # Generated by AI after execution
│   ├── 2-03-25-parent-task/
│   │   ├── task.md
│   │   ├── task.json
│   │   ├── 3-03-25-subtask-one/
│   │   │   ├── task.md
│   │   │   ├── task.json
│   │   │   ├── memory-1.md         # Generated by AI (execution 1)
│   │   │   ├── memory-2.md         # Generated by AI (execution 2)
│   │   │   ├── feedback-1.md       # Human feedback (after execution 1)
│   │   │   └── feedback-2.md       # Human feedback (after execution 2)
│   │   └── 4-03-25-subtask-two/
│   │       ├── task.md
│   │       └── task.json
│   └── archive/                    # Archived tasks (by year-month)
│       └── 2026-03/
│           └── ...
├── logs/                           # Execution logs
│   ├── fa.log                      # Tool internal logs
│   └── agents/                     # Agent execution logs
│       └── 1-03-25-first-task/
│           ├── round-1-iflow.log
│           └── round-2-iflow.log
├── policies/                       # Policy definitions
│   ├── testing.yml
│   └── security.yml
└── templates/                      # Jinja2 templates (optional overrides)
    └── task_prompt.j2
```

## Auto-initialization

Commands automatically create `.fa/` structure on-demand:

- `.fa/tasks/`
- `.fa/tasks/archive/`
- `.fa/logs/`
- `.fa/logs/agents/`
- `.fa/policies/`
- `.fa/templates/`
